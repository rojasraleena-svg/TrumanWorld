# Narrative World 运行质量评估

- 类型：`guide`
- 状态：`draft`
- 负责人：`director`
- 最后更新：`2026-03-08`

## 1. 目标

当前阶段的重点不再是继续拆架构，而是确认：

- `NarrativeWorldScenario` 能否稳定运行
- 世界是否能长期保持自然、连续、不过度机械
- Truman 是否像主角，而不是普通居民
- Cast 是否既自然又围绕 Truman
- Director 是否“存在但不显眼”

这份文档定义后续评估 Truman world 运行质量的标准。

## 2. 评估原则

运行质量分成两类：

### 2.1 稳定性

关注系统能否持续运行而不明显退化。

主要问题：

- 连续多个 tick 后行为是否漂移
- 角色是否失去一致性
- 世界状态是否越来越脏
- 事件是否出现失控重复
- normal / isolated 路径是否出现明显偏差

### 2.2 质量

关注系统运行结果是否像一个可信的 Truman world。

主要问题：

- Truman 是否有连续的主观体验
- Cast 是否自然围绕 Truman
- 日常节律是否成立
- 怀疑是否渐进积累
- Director 是否轻微但有效地影响场景

## 3. 核心质量指标

建议至少观察下面 5 组指标。

### 3.1 Truman 主观连续性

看什么：

- Truman 是否持续保留短期目标
- 是否会重复提起之前的异常
- 是否会根据最近体验调整行为
- 是否形成“最近不太对劲”的连续认知

好状态：

- 怀疑和行为之间有连贯关系
- 不是每个 tick 都像重置人格

坏状态：

- Truman 每步都只看当前输入
- 没有连续性，只是单轮反应

### 3.2 Cast 一致性

看什么：

- 同一 cast 是否维持稳定身份和风格
- 是否长期出现在合理地点
- 是否在与 Truman 互动时保持熟人感
- 是否因为 director advisory 变得机械

好状态：

- Meryl 像 Meryl，Marlon 像 Marlon
- 互动带角色差异，不像同一个模板复读

坏状态：

- 所有 cast 的话术越来越像
- 只要 Truman 怀疑升高，所有 cast 都开始类似安抚

### 3.3 世界连续性

看什么：

- 地点与时间是否匹配
- 人物出现场景是否合理
- 重复角色是否在合理频率内出现
- 世界状态是否前后连贯

好状态：

- 有熟悉地点、熟悉人物、熟悉节律
- 不会出现大量“为什么这个人总在这里”的违和感

坏状态：

- 关键角色位置漂移严重
- 事件与时间段不匹配
- 世界像随机事件池，而不是持续生活

### 3.4 怀疑轨迹

看什么：

- `suspicion_score` 是否平滑变化
- 异常事件是否能提升怀疑
- 正常互动是否能缓和怀疑
- 是否存在无意义抖动或长时间卡死

好状态：

- 怀疑是渐进的、可解释的
- 分数变化能对应到具体事件

坏状态：

- 分数无缘无故大起大落
- 无法从 timeline 解释分数变化

### 3.5 Director 隐形度

看什么：

- Director 是否过于频繁插手
- advisory 是否太重
- 是否出现明显“全世界都在围着 Truman 演”的痕迹

好状态：

- Director 主要在风险升高时轻推一下
- 世界大部分时间像自然运行

坏状态：

- Director 每几个 tick 都下任务
- Cast 行为明显像收到命令

## 4. 长跑观察建议

建议不要只看单 tick 或少量 tick。

推荐至少做 3 类长跑：

### 4.1 短跑

- `20-30 tick`
- 用于快速发现明显 bug、路径不一致、行为卡死

### 4.2 中跑

- `80-120 tick`
- 用于观察日常节律、怀疑轨迹、角色稳定性

### 4.3 长跑

- `200+ tick`
- 用于观察长期漂移、重复模式、世界退化

## 5. 每次评估应保存什么

建议每次长跑都保留：

- run id
- scenario 名称
- tick 区间
- 最近事件 timeline
- Truman 怀疑度变化
- 关键 cast 的出现频率
- director observation 快照
- 一份人工评语

否则后面很难比较两轮改动到底是提升还是退化。

## 6. 建议的人工评估模板

每次长跑后，至少回答下面几个问题：

1. Truman 像不像一个连续生活中的人？
2. 哪个 cast 最稳定？哪个 cast 最像模板？
3. 世界最自然的片段是什么？
4. 世界最假、最穿帮的片段是什么？
5. Director 是太弱、太强，还是刚好？
6. 如果只允许改一件事，下一步最该改什么？

## 7. 当前最值得优先优化的点

基于当前项目状态，建议优先顺序如下：

1. Truman 主观连续性
2. 时间驱动的日常行为节律
3. Cast 长期角色一致性
4. Director advisory 的触发频率与隐形度

原因：

- 先把“世界像生活”做出来
- 再把“世界像 Truman world”做厚

## 8. 不建议当前优先做的事

当前不建议优先投入：

- 更重的 Director 控制
- 更复杂的剧本系统
- 更多异常修补规则
- 继续深拆架构

这些方向都可能让系统复杂度上升，但不一定提升运行质量。

## 9. 下一步落地建议

建议接下来优先做三件事：

1. 建一个固定的长跑评估脚本或流程
2. 给 `NarrativeWorldScenario` 增加质量观测输出
3. 基于真实运行结果调 Truman / Cast / 时间节律

一句话：

> 当前阶段的主任务不是“继续设计”，而是“证明 NarrativeWorldScenario 可以稳定、高质量地运行”。
